VCV Rack Patch Prover

How the signal graph and audio reachability system is built

What problem this solves

A VCV Rack patch is a graph of modules wired together with cables. Whether that patch actually produces audio depends on two conditions:

  1. Existence — every module in the patch is installed on this machine.
  2. Reachability — there is a continuous audio path from at least one sound source to the audio output (AudioInterface2).

Both of these can fail silently. A missing module is quietly skipped by VCV Rack. A broken cable chain produces no error — just silence. This system proves both conditions from the patch file alone, before opening VCV Rack.

Key design principle: provability is a property of the graph, not a separate step. After construction, graph.audio_reachable is always current — no explicit "prove" call exists.

Layer 1: Node base classes node.py

Every module type is a subclass of Node. The key contract is a single abstract method: 

def audio_out_for(self, audio_in_ports: frozenset[int]) -> frozenset[int]:
    """
    Given which input port IDs currently carry audio,
    return which output port IDs emit audio.
    """

This is what each module class knows about itself. The graph calls it during propagation; no module needs to know about the rest of the patch.

AudioSourceNode ── always emits audio (oscillators, samplers, drum voices) AudioProcessorNode ── routes specific inputs to specific outputs (VCF, VCA, Reverb) AudioMixerNode ── any audio input activates all audio outputs (Mix4, Mix8, mscHack) AudioSinkNode ── terminal consumer, no audio outputs (AudioInterface2) ControllerNode ── CV / Gate / Clock only, never passes audio (ADSR, LFO, Clock) UnknownNode ── module not in our registry; conservative all-to-all passthrough

AudioProcessorNode — explicit route declaration

Processor subclasses declare _routes: a list of (input_port_id, output_port_id) pairs. Audio flows from an input to its paired output only when that input port receives audio. 

class VCFNode(AudioProcessorNode):
    PLUGIN = "Fundamental"
    MODEL  = "VCF"
    # IN (port 3) → LPF (port 0), HPF (port 1)
    _routes = [(3, 0), (3, 1)]

UnknownNode — the conservative assumption

If a module isn't in our registry, it gets UnknownNode. Its audio_out_for returns a large sentinel set covering ports 0–63. The graph then constrains that to only actually-connected ports, so an unknown module never silently blocks a proven audio path.

All classes self-register in NODE_REGISTRY, a dict mapping "plugin/model" strings to classes. PatchLoader looks up each module in this dict; anything missing becomes an UnknownNode.

Layer 3: SignalGraph signal_graph.py

SignalGraph owns a dict of Nodes and a list of Edges (one per cable). Its properties are computed on demand from current topology.

Forward propagation — the core algorithm

Propagation runs iteratively until no more audio ports are discovered (fixed point). Because audio sets can only grow and are bounded by the finite port count, this always terminates — even in the presence of feedback loops. 

def _propagate(self) -> dict[int, frozenset[int]]:
    # audio_in[module_id] = set of input ports that carry audio
    audio_in = {mid: set() for mid in self._nodes}
    changed = True

    while changed:
        changed = False
        for edge in self._edges:
            src = self._nodes[edge.src_node]
            src_audio_out = src.audio_out_for(frozenset(audio_in[edge.src_node]))

            if edge.src_port in src_audio_out:
                if edge.dst_port not in audio_in[edge.dst_node]:
                    audio_in[edge.dst_node].add(edge.dst_port)
                    changed = True

    return {mid: frozenset(ports) for mid, ports in audio_in.items()}

AudioSourceNode subclasses return their audio outputs regardless of input, so they seed the propagation automatically — no special "seed sources" step needed.

Layer 4: InstalledRegistry installed.py

Scans ~/Library/Application Support/Rack2/plugins-mac-arm64/ (and fallback paths for other platforms) for plugin.json files, collecting all plugin/model pairs. Loading is lazy and cached.

Core modules (AudioInterface2, MIDIToCVInterface, etc.) are built into VCV Rack itself — no plugin.json exists for them. They are hardcoded in BUILTIN and always treated as installed. This was a real bug we hit: AudioInterface2 was reported missing until we added the hardcoded set. 
# Usage
reg = InstalledRegistry.default()   # singleton, lazy-loaded
reg.has("Fundamental", "VCO")       # True
reg.has("Autodafe", "ClockDivider")  # False on ARM64 Mac

Layer 5: PatchLoader loader.py

The only layer that reads raw .vcv JSON. A .vcv file is a zstandard-compressed POSIX tar archive containing patch.json. 

# patch.json structure (simplified)
{
  "modules": [
    { "id": 12345, "plugin": "Fundamental", "model": "VCO",
      "params": [{ "id": 1, "value": 0.0 }] }
  ],
  "cables": [
    { "outputModuleId": 12345, "outputId": 3,
      "inputModuleId": 67890, "inputId": 0 }
  ]
}

For each module, PatchLoader looks up NODE_REGISTRY and instantiates the matching class. Unknown modules become UnknownNode. Each cable becomes an Edge(src_node, src_port, dst_node, dst_port).

The proof in action

The demo patch tests/square_to_output.py:

# Build the patch
patch = Patch()
vco   = patch.add("Fundamental", "VCO",           pos=[0, 0])
audio = patch.add("Core",        "AudioInterface2", pos=[8, 0])
patch.connect(vco.SQR, audio.i.IN_L)
patch.connect(vco.SQR, audio.i.IN_R)
patch.save("tests/square_to_output.vcv")

# Prove it
graph = PatchLoader.load("tests/square_to_output.vcv")
assert not graph.missing_modules()   # step 1: existence
assert graph.audio_reachable          # step 2: reachability

Output:

Nodes: 2 Edges: 2 All modules installed: yes AUDIO REACHABLE: yes Audio chain: Fundamental/VCO#7109571174881621 Core/AudioInterface2#4329850745837336 All assertions passed.

A patch that would fail existence

Nodes: 2 Edges: 2 MODULES MISSING (1) -- cannot prove reachability: ! Autodafe/ClockDivider AUDIO REACHABLE: unknown (missing modules)

Existence failure short-circuits the report — there is no point checking reachability if a module won't load.

Known limitations and next steps

  • Port numbers are hand-declared. They come from reading each plugin's C++ source or observing live patches. A wrong port number produces a false negative (audio declared unreachable when it isn't). This happened with Chronoblob2 (audio inputs are 5, 6 — not 0, 1) and Pressor (audio outputs are 1, 2 — not 0, 1).
  • Params are not evaluated. A VCA with its level knob at zero will pass the reachability check but produce silence. The warnings property flags obvious zero-level cases, but this is advisory only.
  • UnknownNode is optimistic. Modules not in our registry are assumed to pass audio. Adding more module classes makes the proof tighter.
  • Signal type is not tracked. The graph propagates a single "audio present" boolean per port. A future version could track signal type (audio, CV, gate, clock) and flag cables that carry the wrong type into an input.
